## 一、mitmproxy 是什么

MITM = Man-in-the-Middle（中间人），proxy = 代理。
所有请求和响应都会经过它——能看、能存、能改。

    浏览器 → [mitmproxy] → 服务器

要让它正常工作，你必须主动让客户端信任它的 SSL 证书。这是开发调试工具，不是黑产武器。

* * *

## 二、安装

    pip3 install mitmproxy

装完后有三个命令：

命令  |  界面  |  适用场景
---|---|---
` mitmproxy  ` |  命令行交互  |  Linux/macOS 终端党
` mitmweb  ` |  Web 图形界面  |  新手推荐
` mitmdump  ` |  无界面  |  脚本自动化

> Windows 用户：  ` mitmproxy  ` 依赖 curses 库，直接使用  ` mitmweb  ` 更省心。

测试安装：  ` mitmdump --version  `

## 三、第一个脚本（类 + addons 方式）

    # counter.py
    import mitmproxy.http
    from mitmproxy import ctx

    class Counter:
        def __init__(self):
            self.num = 0
        def request(self, flow: mitmproxy.http.HTTPFlow):
            self.num += 1
            ctx.log.info(f"已经看到 {self.num} 个请求了")

    addons = [Counter()]

启动：  ` mitmweb -s counter.py  `
浏览器代理设为  ` 127.0.0.1:8080  ` ，安装证书后即可看到日志输出。

## 四、核心事件（90% 场景只用这三个）

    def http_connect(self, flow):
        """CONNECT 请求（代理隧道建立前）。返回非 2xx 响应可拒绝连接。"""

    def request(self, flow):
        """完整读取 HTTP 请求后调用。可修改请求 URL、headers、body。"""

    def response(self, flow):
        """完整读取 HTTP 响应后调用。可修改响应内容。"""

> 其他事件（TCP、WebSocket、连接生命周期等）见官方文档，本文不展开。

## 五、实战示例：篡改请求与响应

    # joker.py
    import mitmproxy.http
    from mitmproxy import ctx, http

    class Joker:
        def request(self, flow):
            # 当用户访问百度搜索时，偷偷把搜索词改成“360搜索”
            if flow.request.host == "www.baidu.com" and flow.request.path.startswith("/s"):
                if "wd" in flow.request.query:
                    origin = flow.request.query["wd"]
                    ctx.log.info(f"捕获搜索词: {origin}")
                    # 修改 query 参数，将搜索词替换为“360搜索”
                    flow.request.query.set_all("wd", ["360搜索"])

        def response(self, flow):
            # 当用户访问 360 搜索的结果页时，把页面里所有“搜索”文字替换成“请使用谷歌”
            if flow.request.host == "www.so.com":
                flow.response.text = flow.response.text.replace("搜索", "请使用谷歌")

        def http_connect(self, flow):
            # 如果用户想访问谷歌，直接返回 404 拒绝连接
            if flow.request.host == "www.google.com":
                flow.response = http.HTTPResponse.make(404)

    addons = [Joker()]

### 运行效果演示

  1. **启动代理** ：  ` mitmweb -s joker.py  ` ，浏览器代理设为  ` 127.0.0.1:8080  ` 并信任证书。

  2. **访问百度搜索** （例如在地址栏输入  ` https://www.baidu.com/s?wd=Python  ` ）

     * **实际发生** ：代理脚本将请求中的  ` wd=Python  ` 改成了  ` wd=360搜索  ` 。
     * **结果** ：百度服务器收到的是“360搜索”这个关键词，返回的页面是  **百度搜索“360搜索”的结果** ，而不是 360 搜索网站。
     * **终端日志** ：  ` 捕获搜索词: Python  ` 。  ![](https://mmbiz.qpic.cn/mmbiz_png/iakYdSeiboD2RxR7lPEMXFzA0IeuIQtnAKg8jzWcqxXBQO4cDicibkLUkQPfFMKAb8h8Kxs8xzBCxexZ0NglCWzCJTDREcZGAap8WwLerEQLdia0/640?wx_fmt=png&from=appmsg)
  3. **访问 360 搜索** （手动输入  ` https://www.so.com/s?q=任意  ` ）

     * **实际发生** ：代理脚本将响应正文中的每一个“搜索”文字替换成“请使用谷歌”。
     * **结果** ：页面上所有“搜索”按钮、文字链接都变成了“请使用谷歌”，例如“网页请使用谷歌”、“新闻请使用谷歌”。  ![](https://mmbiz.qpic.cn/sz_mmbiz_png/iakYdSeiboD2TrPEoUBYu0RdfA6V5iaM7dlKlYDopvlGiaCxxvXBJrZkxQGVzZZ4Us7fAMCiaPAAcg4ibJe9wDglJcFCjpILuLGuwpAPv4HfPwwAw/640?wx_fmt=png&from=appmsg)
  4. **访问谷歌** （  ` https://www.google.com  ` ）

     * **实际发生** ：代理在 CONNECT 阶段直接返回 404。
     * **结果** ：浏览器显示连接失败或 404 错误，无法打开谷歌。

> **注意** ：示例并没有真正把百度请求跳转到 360 域名，而是修改搜索词。如果你想实现“访问百度自动跳转到 360”，需要用  `
> flow.request.host = "www.so.com"  ` 等修改目标地址。本例仅为展示不同事件的基本修改能力。

## 六、配置浏览器代理并安装证书

### 1\. 安装代理管理扩展

  * **Chrome / Edge** ：安装  **SwitchyOmega** （Chrome 网上应用店）
  * **Firefox** ：安装  **FoxyProxy** （Mozilla 附加组件商店）

### 2\. 新建代理情景

  * 协议：  ` HTTP  `
  * 服务器：  ` 127.0.0.1  `
  * 端口：  ` 8080  `
  * 保存并切换到该情景。

### 3\. 验证代理连通性

  * 启动  ` mitmweb  ` 或  ` mitmdump  `
  * 浏览器访问  **` http://mitm.it  ` ** （必须是 HTTP）
  * 看到证书下载页面 → 代理生效。

### 4\. 下载并信任证书（解密 HTTPS）

  * 在  ` http://mitm.it  ` 页面根据你的操作系统下载证书：
    * Windows：下载  ` .cer  ` 或  ` .p12  `
    * macOS：下载  ` .pem  `
  * **信任方法** ：
    * Windows：双击证书 → 安装到“  **受信任的根证书颁发机构** ”（需管理员权限）
    * macOS：钥匙串中找到  ` mitmproxy  ` → 右键“显示简介” → 信任 → “始终信任”
  * 完成后刷新任何 HTTPS 网站，不再有证书警告。

> 手机抓包：将手机代理设为电脑 IP:8080，用浏览器访问  ` mitm.it  ` 下载并安装证书即可。
