## 先看问题

打开  码上爬题六  [1]  ，要求很简单：20 页数据，每页 10 个数字，把 200 个数字加起来，提交答案。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/iakYdSeiboD2TTQqHpow4guFOCaayVMxoYTib5eETX5xMbzJaRgxhBicmgQSjuibfhHVgvickCYkkHzeNjgctW895MoLY1ZyQRDVWl15O1y9KGCxQ/640?wx_fmt=png&from=appmsg)

F12 → Network，刷新，第一个数据请求就暴露了底牌。

请求头多了两个东西：

    s: d7f163efdb2279fbdee50873d47fb843
    tt: 1779686528321

` tt  ` 是 13 位毫秒时间戳，  ` s  ` 是 32 位 hex——标准 MD5 特征。

响应的 body 也不正常，不是 JSON，是一串 hex 字符串。说明响应体也加密了。

问题拆开就是两个：

  1. **请求签名怎么算？** → 生成 s 和 tt 的规则
  2. **响应怎么解密？** → 把 hex 还原成 JSON

* * *

## 签名怎么算

定位加密入口，最直接的办法：通过请求的调用堆栈回溯，从第一个堆栈  ` pagination6.js  ` 我们就发现了问题。

![](https://mmbiz.qpic.cn/mmbiz_png/iakYdSeiboD2TpJp1sr04icmoXwUPgT31RQ3gLpfpz7eq4dZ6BMQQHiaaWzXyuOaHHuelHZeszE8uuQcX1JnP0YdVbiciaBNibERb88uGCiahqcY6g4/640?wx_fmt=png&from=appmsg)

开头是一个 MD5 实现，暴露了  ` window.xxoo  ` 。往下翻，签名函数在这：

    function s() {
     window.ttt = new Date().getTime();
     window.token = window.xxoo("sssssbbbbb" + ttt);
     window.hhh = { s: window.token, tt: window.ttt };
    }

一句话：  **s = MD5("sssssbbbbb" + 时间戳)** 。

盐值  ` sssssbbbbb  ` 直接硬编码在源码里。不是从服务器取的，不是动态生成的，就写在那一行。这种属于送了。

* * *

## 响应怎么解密

翻到响应加密函数：

    let kkkk = dd.a.enc.Utf8.parse("xxxxxxxxoooooooo");
    let iiii = dd.a.enc.Utf8.parse("0123456789ABCDEF");

    function xxxxoooo(encryptedHex) {
    let enccc = dd.a.enc.Hex.parse(encryptedHex);
    let deccc = dd.a.AES["decr" + "ypt"]({ciphertext: enccc}, kkkk, {
    mode: dd.a.mode.CBC,
    padding: dd.a.pad.Pkcs7,
    iv: iiii,
     });
    return deccc.toString(dd.a.enc.Utf8);
    }

参数一览：

参数  |  值
---|---
算法  |  AES-CBC
密钥  |  ` xxxxxxxxoooooooo  ` （16 字节）
IV  |  ` 0123456789ABCDEF  ` （16 字节）
填充  |  PKCS7

` "decrypt"  ` 被拆成  ` "decr" + "ypt"  ` ，一个小技巧——防止你直接搜  ` decrypt  ` 定位。但你搜  `
parse(  ` 也能找到，防不了谁。

## 踩了一个坑

写完脚本跑，20 页全部报错：

    unsupported operand type(s) for +: 'int' and 'str'

打印数据一看，返回的不是数字列表：

    "请登录以便生成数组完成计算..."

**原因** ：API 靠  ` sessionid  ` cookie 鉴定身份。没有这个 cookie，服务端给你一段提示文字。

解决：浏览器登录后，Application → Cookies 里把  ` sessionid  ` 拷出来，写进  ` cookie.txt  `
，脚本启动时读一下就行。

* * *

## 完整脚本

    """
    题六：倚天剑 - JS逆向破解脚本
    网站: https://www.mashangpa.com/problem-detail/6/

    加密分析:
    1. 请求签名: MD5("sssssbbbbb" + timestamp)
    2. 响应解密: AES-CBC, key="xxxxxxxxoooooooo", iv="0123456789ABCDEF"

    用法: 将浏览器登录后的 sessionid cookie 写入同目录下的 cookie.txt，格式:
      sessionid=xxxx
    """
    import hashlib
    import json
    import os
    import time
    import requests
    from Crypto.Cipher import AES
    from Crypto.Util.Padding import unpad

    BASE_URL = "https://www.mashangpa.com"
    PROBLEM_ID = 6
    AES_KEY = b"xxxxxxxxoooooooo"# 16 bytes
    AES_IV = b"0123456789ABCDEF"# 16 bytes
    SALT = "sssssbbbbb"

    def load_cookie():
        """从 cookie.txt 加载登录 cookie"""
        path = os.path.join(os.path.dirname(__file__), "cookie.txt")
        if os.path.exists(path):
            with open(path) as f:
                for line in f:
                    line = line.strip()
                    if "=" in line and not line.startswith("#"):
                        return line
        return None

    def generate_signature() -> tuple:
        """生成请求签名: MD5(盐值 + 毫秒时间戳)"""
        timestamp = int(time.time() * 1000)
        signature = hashlib.md5((SALT + str(timestamp)).encode()).hexdigest()
        return signature, timestamp

    def decrypt_response(encrypted_hex: str) -> dict:
        """AES-CBC 解密服务端响应"""
        cipher = AES.new(AES_KEY, AES.MODE_CBC, iv=AES_IV)
        encrypted_bytes = bytes.fromhex(encrypted_hex)
        decrypted = unpad(cipher.decrypt(encrypted_bytes), AES.block_size)
        return json.loads(decrypted.decode("utf-8"))

    def fetch_page(session: requests.Session, page: int) -> dict:
        """请求单页数据并解密"""
        s, tt = generate_signature()
        headers = {"s": s, "tt": str(tt)}

        url = f"{BASE_URL}/api/problem-detail/{PROBLEM_ID}/data/"
        resp = session.get(url, params={"page": page}, headers=headers)
        resp.raise_for_status()

        data = resp.json()
        return decrypt_response(data["t"])

    def main():
        session = requests.Session()

        cookie = load_cookie()
        if cookie:
            session.headers["Cookie"] = cookie
        else:
            print("[!] 未找到 cookie.txt，先访问页面建立会话...")

        session.get(f"{BASE_URL}/problem-detail/{PROBLEM_ID}/")

        total_sum = 0
        all_numbers = []

        for page in range(1, 21):
            decrypted = fetch_page(session, page)
            current_array = decrypted.get("current_array", [])

            if isinstance(current_array, str):
                print(f"[!] 第{page}页返回了字符串而非数组: {current_array}")
                print("[!] 请先登录 https://www.mashangpa.com/login/")
                print("[!] 登录后从 F12 → Application → Cookies 复制 sessionid 的值")
                print("[!] 写入本目录的 cookie.txt: sessionid=你的值")
                return

            page_sum = sum(current_array)
            total_sum += page_sum
            all_numbers.extend(current_array)
            print(f"第{page:2d}页: 数组={current_array}, 本页和={page_sum}, 累计={total_sum}")

        print(f"\n{'='*50}")
        print(f"总计页数: 20, 元素总数: {len(all_numbers)}")
        print(f"所有元素之和(答案): {total_sum}")
        print(f"{'='*50}")

        return total_sum

    if __name__ == "__main__":
        main()

## 完

这道题放在 JS 逆向里算简单那一档。没有什么动态混淆，没有环境检测，没有运行时解密。盐值写在源码里，密钥写在源码里，IV 也写在源码里。

但不是每道题都这么友善。很多加密入口藏得深——可能藏在 Webpack 的 chunk 里，可能经过多层
eval，甚至加密函数是动态从后端拉下来的。那就得走另一套更麻烦的流程了。

不过那就是另外的价钱了。

要学什么  |  原因
---|---
MD5 签名识别  |  32 位 hex + 时间戳是标准组合

Cookie 传递  |  有些接口不鉴权只挡自动化，拷 cookie 就行
