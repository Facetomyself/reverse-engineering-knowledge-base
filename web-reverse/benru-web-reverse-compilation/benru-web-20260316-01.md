在JS逆向中，字符串混淆是最常见的保护手段之一。网站会把关键的API地址、加密密钥、错误提示等字符串进行编码或拆分，让你在代码里看不到明文。比如：

    // 十六进制编码
    var url = '\x68\x74\x74\x70\x73\x3a\x2f\x2f\x61\x70\x69\x2e\x65\x78\x61\x6d\x70\x6c\x65\x2e\x63\x6f\x6d';

    // Unicode编码
    var key = '\u0041\u0042\u0043\u0044';

    // 数组拆分
    var arr = ['https://', 'api.', 'example', '.com'];
    var url = arr[0] + arr[1] + arr[2] + arr[3];

这些混淆虽然让人眼花缭乱，但通过AST（抽象语法树）可以轻松还原。今天我们就一步步学习如何用AST还原各种字符串混淆。

* * *

## 一、字符串混淆的常见形式

在动手之前，先了解敌人长什么样：

### 1\. 十六进制编码

    var str = '\x48\x65\x6c\x6c\x6f';  // 对应 "Hello"

### 2\. Unicode编码

    var str = '\u0048\u0065\u006c\u006c\u006f';  // 对应 "Hello"

### 3\. 字符串拼接

    var str = 'He' + 'll' + 'o';

### 4\. 数组+索引拼接

    var parts = ['Hel', 'lo', ' ', 'World'];
    var str = parts[0] + parts[1] + parts[2] + parts[3];

### 5\. 自执行函数解密

    var str = (function(){ return 'Hello' })();

### 6\. 复杂的解密函数（比如用移位、异或）

    function decode(s) {
      return s.split('').map(c => String.fromCharCode(c.charCodeAt(0) ^ 0x1)).join('');
    }
    var str = decode('Ifmmp');  // 对应 "Hello"

对于前4种，用AST可以轻松还原；第5种可能需要计算常量表达式；第6种则需要模拟执行或静态分析。本文重点介绍前几种，让你快速上手。

## 三、还原十六进制和Unicode编码

在AST中，字符串节点是  ` StringLiteral  ` 。无论是十六进制还是Unicode，解析器已经帮我们解码了，节点的  ` value  `
属性就是真实的字符串。但是，混淆工具通常会在  ` extra  ` 属性里保留原始编码形式，导致重新生成代码时还是编码后的样子。

所以，还原的关键是  **删除  ` extra  ` 属性** ，这样生成器就会用  ` value  ` 来生成代码。

### 代码示例

    const parser = require('@babel/parser');
    const traverse = require('@babel/traverse').default;
    const generate = require('@babel/generator').default;

    const code = `
    var url = '\\x68\\x74\\x74\\x70\\x73\\x3a\\x2f\\x2f\\x61\\x70\\x69\\x2e\\x65\\x78\\x61\\x6d\\x70\\x6c\\x65\\x2e\\x63\\x6f\\x6d';
    var key = '\\u0041\\u0042\\u0043\\u0044';
    `;

    const ast = parser.parse(code);

    traverse(ast, {
      StringLiteral(path) {
        // 删除 extra 属性，强制用 value 生成代码
        delete path.node.extra;
      }
    });

    const output = generate(ast).code;
    console.log(output);

运行结果：

    var url = 'https://api.example.com';
    var key = 'ABCD';

是不是很简单？只需三行代码，就把编码字符串还原了。

## 四、还原字符串拼接

对于  ` 'He' + 'll' + 'o'  ` 这种常量拼接，我们可以计算其结果，用一个字符串节点替换整个  ` +  ` 表达式。

### 思路

遍历  ` BinaryExpression  ` （二元表达式），如果运算符是  ` +  `
，并且左右两边都是字符串字面量（或者是经过计算的字符串），就合并它们。重复这个过程直到无法合并。

### 代码示例

    const t = require('@babel/types');

    traverse(ast, {
      // 从内向外遍历，优先处理子节点
      BinaryExpression: {
        exit(path) {
          const { node } = path;
          if (node.operator !== '+') return;

          // 如果左右都是字符串字面量
          if (t.isStringLiteral(node.left) && t.isStringLiteral(node.right)) {
            let value = node.left.value + node.right.value;
            path.replaceWith(t.stringLiteral(value));
          }
          // 如果左边是字符串字面量，右边是字符串拼接的结果（已经变成了字符串字面量）
          else if (t.isStringLiteral(node.left) && t.isStringLiteral(node.right)) {
            // 实际上上面的条件已经覆盖，因为遍历退出时子节点已经被替换
          }
          // 可以扩展处理更多情况，比如字符串 + 数字等
        }
      }
    });

注意：我们使用了  ` exit  ` 阶段，确保子节点先被处理。这样可以从内到外合并所有连续的  ` +  ` 。

测试代码：

    const code = `
    var str = 'He' + 'll' + 'o' + ' ' + 'World';
    `;

经过处理后，会变成：

    var str = 'Hello World';

## 五、还原数组+索引拼接

这是更常见的混淆方式：定义一个数组，里面放字符串片段，然后通过下标拼接。例如：

    var parts = ['https://', 'api.', 'example', '.com'];
    var url = parts[0] + parts[1] + parts[2] + parts[3];

还原思路：

  1. 找到数组定义，记录每个元素的值。
  2. 找到对数组元素的引用，如  ` parts[0]  ` ，将其替换为对应的字符串字面量。
  3. 然后应用上面的字符串拼接还原。

### 代码示例

    traverse(ast, {
      // 先收集数组定义
      VariableDeclarator(path) {
        const { id, init } = path.node;
        if (t.isIdentifier(id) && t.isArrayExpression(init)) {
          // 将数组元素的值保存到一个 Map 中
          const arrayName = id.name;
          const elements = init.elements.map(elem => {
            if (t.isStringLiteral(elem)) return elem.value;
            // 如果元素不是字符串字面量，可以保留原样，或者尝试进一步计算
            return null;
          });
          // 将这个数组信息附加到当前作用域（这里简化，用全局 Map）
          if (!global.arrays) global.arrays = new Map();
          global.arrays.set(arrayName, elements);
        }
      },

      // 替换数组成员访问
      MemberExpression(path) {
        const { object, property } = path.node;
        if (t.isIdentifier(object) && t.isNumericLiteral(property)) {
          const arrayName = object.name;
          const index = property.value;
          if (global.arrays && global.arrays.has(arrayName)) {
            const elements = global.arrays.get(arrayName);
            if (index >= 0 && index < elements.length && elements[index] !== null) {
              // 用字符串字面量替换成员表达式
              path.replaceWith(t.stringLiteral(elements[index]));
            }
          }
        }
      }
    });

    // 然后再进行字符串拼接还原（需要再次遍历）
    traverse(ast, {
      BinaryExpression: {
        exit(path) {
          // 同上
        }
      }
    });

这个脚本会先收集数组信息，然后把  ` parts[0]  ` 之类的表达式替换为实际的字符串，最后合并拼接。最终  ` url  ` 变量会变成  `
'https://api.example.com'  ` 。

注意：实际混淆中，数组可能被多个地方引用，或者数组元素本身也是表达式。这里只是一个简单示例，更复杂的情况需要更多处理。

## 六、综合案例：还原一个真实的混淆字符串

假设我们有如下混淆代码：

    var _0x4b8c = ['\x68\x74\x74\x70\x73\x3a\x2f\x2f', '\x61\x70\x69\x2e', '\x65\x78\x61\x6d\x70\x6c\x65', '\x2e\x63\x6f\x6d'];
    var url = _0x4b8c[0] + _0x4b8c[1] + _0x4b8c[2] + _0x4b8c[3];
    var key = '\x41\x42\x43\x44';

我们希望还原出：

    var url = 'https://api.example.com';
    var key = 'ABCD';

我们可以组合上面学到的方法，写出一个完整的还原脚本：

    const parser = require('@babel/parser');
    const traverse = require('@babel/traverse').default;
    const generate = require('@babel/generator').default;
    const t = require('@babel/types');

    const code = `
    var _0x4b8c = ['\\x68\\x74\\x74\\x70\\x73\\x3a\\x2f\\x2f', '\\x61\\x70\\x69\\x2e', '\\x65\\x78\\x61\\x6d\\x70\\x6c\\x65', '\\x2e\\x63\\x6f\\x6d'];
    var url = _0x4b8c[0] + _0x4b8c[1] + _0x4b8c[2] + _0x4b8c[3];
    var key = '\\x41\\x42\\x43\\x44';
    `;

    const ast = parser.parse(code);

    // 第一步：收集数组
    const arrays = new Map();

    traverse(ast, {
      VariableDeclarator(path) {
        const { id, init } = path.node;
        if (t.isIdentifier(id) && t.isArrayExpression(init)) {
          const elements = init.elements.map(elem => {
            if (t.isStringLiteral(elem)) {
              // 删除 extra，保证值是真实的
              delete elem.extra;
              return elem.value;
            }
            return null;
          });
          arrays.set(id.name, elements);
        }
      }
    });

    // 第二步：替换数组成员访问
    traverse(ast, {
      MemberExpression(path) {
        const { object, property } = path.node;
        if (t.isIdentifier(object) && t.isNumericLiteral(property)) {
          const arrayName = object.name;
          const index = property.value;
          if (arrays.has(arrayName)) {
            const elements = arrays.get(arrayName);
            if (index >= 0 && index < elements.length && elements[index] !== null) {
              path.replaceWith(t.stringLiteral(elements[index]));
            }
          }
        }
      }
    });

    // 第三步：还原十六进制/Unicode字符串（删除extra）
    traverse(ast, {
      StringLiteral(path) {
        delete path.node.extra;
      }
    });

    // 第四步：合并字符串拼接
    traverse(ast, {
      BinaryExpression: {
        exit(path) {
          const { node } = path;
          if (node.operator !== '+') return;

          // 尝试合并两个字符串字面量
          if (t.isStringLiteral(node.left) && t.isStringLiteral(node.right)) {
            const value = node.left.value + node.right.value;
            path.replaceWith(t.stringLiteral(value));
          }
          // 可以继续处理更多情况，如字符串+数字等
        }
      }
    });

    // 最后，再次遍历删除可能新产生的extra（上一步新建的字符串字面量也可能有extra）
    traverse(ast, {
      StringLiteral(path) {
        delete path.node.extra;
      }
    });

    const output = generate(ast).code;
    console.log(output);

运行后，输出：

    var _0x4b8c = ['https://', 'api.', 'example', '.com'];
    var url = 'https://api.example.com';
    var key = 'ABCD';

虽然数组本身还是保留了编码（因为我们没有处理数组元素的extra），但  ` url  `
已经变成完整字符串。如果我们还想清理数组，可以再对数组元素做一次删除extra。

## 七、进阶：处理动态解密函数

有时候字符串不是直接给出，而是通过一个函数动态计算。比如：

    function decode(s) {
      return s.split('').map(c => String.fromCharCode(c.charCodeAt(0) ^ 0x1)).join('');
    }
    var str = decode('Ifmmp');

这种还原需要模拟执行函数，或者用静态分析推导结果。对于新手，可以先跳过，或者利用Node的  ` vm  `
模块在沙箱中执行代码片段来获取结果。但这涉及安全问题，建议只在可控环境中使用。

简单示例（不安全，仅演示）：

    const vm = require('vm');
    const code = `
    function decode(s) {
      return s.split('').map(c => String.fromCharCode(c.charCodeAt(0) ^ 0x1)).join('');
    }
    var str = decode('Ifmmp');
    `;
    const sandbox = {};
    vm.createContext(sandbox);
    vm.runInContext(code, sandbox);
    console.log(sandbox.str); // 输出 "Hello"

但这种方法有风险，如果代码中有恶意操作可能会危害系统。更稳妥的方式是使用AST进行符号执行，不过那比较复杂，以后有机会再介绍。

## 总结

用AST还原字符串混淆其实并不难，核心就是：

  * 找到字符串节点，删除  ` extra  ` 让它们以明文显示。
  * 对于拼接，合并常量字符串。
  * 对于数组引用，替换为实际元素值。

掌握了这些技巧，你已经能对付大部分字符串混淆了。下一期我们聊聊如何还原控制流平坦化，敬请期待！
