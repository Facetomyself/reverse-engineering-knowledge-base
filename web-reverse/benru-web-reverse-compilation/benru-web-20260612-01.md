## 二、请求头参数  ` m  ` 的生成逻辑

反混淆后，找到设置请求头的关键代码：

    let v66 = new Date().getTime();          // 毫秒时间戳
    let v67 = window.eeee("xialuo" + v66);  // 调用全局加密函数
    p926.headers.m = v67;

### 分析  ` window.eeee  ` 函数

搜索  ` window.eeee  ` 的定义，发现它是函数  ` f13  ` 的包装。当只传入一个参数时，逻辑简化为：

    window.eeee = function(p811, p812, p813) {
        // ... 其他分支
        return (function(p839) {
            return f10(f12(p839));
        })(p811);
    };

继续跟踪  ` f12  ` →  ` f7  ` →  ` f8  ` ，最终发现底层实现的是  **标准 MD5 哈希**
，输出为小写十六进制字符串。

因此：

    m = MD5("xialuo" + 当前毫秒时间戳)

### Python 还原

    import hashlib
    import time

    def generate_m():
        ts = int(time.time() * 1000)          # 毫秒时间戳
        m = hashlib.md5(f"xialuo{ts}".encode()).hexdigest()
        return m, ts

    m, ts = generate_m()
    print(f"m: {m}")
    print(f"ts: {ts}")

输出示例：

    m: 5d41402abc4b2a76b9719d911017c592
    ts: 1749744000000

## 三、URL 请求参数  ` x  ` 的生成逻辑

继续查看拼接 URL 的代码：

    p926.url += "&x=" + encodeURIComponent(dd.a.SHA256(v67 + "xxoo"));

  * ` v67  ` 就是上一步生成的  ` m  `
  * 固定字符串  ` "xxoo"  `
  * 调用  ` CryptoJS.SHA256  ` 计算哈希，得到十六进制字符串
  * 最后进行  ` encodeURIComponent  ` （实际十六进制字符无需编码，但保留原风格）

所以：

    x = SHA256( m + "xxoo" )

### Python 还原

    import hashlib
    import urllib.parse

    def generate_x(m):
        x_raw = hashlib.sha256((m + "xxoo").encode()).hexdigest()
        x = urllib.parse.quote(x_raw)   # 保持与前端一致
        return x

    x = generate_x(m)
    print(f"x: {x}")

输出示例：

    x: 7b3d979ca8330a940a7f7c9e2f0c6f0e6e2b5c9a8f4e1d2b3a4c5d6e7f8a9b0c

## 四、响应数据  ` r  ` 的解密逻辑

服务端返回的 JSON 中包含字段  ` r  ` ，其值为一段  **十六进制字符串** 。前端通过 AES-CBC 模式解密。

### 响应拦截器代码

    $.addResponseInterceptor(function (p927, p928, p929) {
        if (p929.statusText === "OK" || p929.statusText === "success") {
            const v68 = f14(p927.r);      // 解密
            const v69 = JSON.parse(v68);  // 解析 JSON
            return v69;
        }
        return p927;
    });

### 解密函数  ` f14  ` 分析

    function f14(p850) {
        let v60 = CryptoJS.enc.Hex.parse(p850);
        const vO17 = { ciphertext: v60 };
        let v61 = CryptoJS.AES.decrypt(vO17, v58, {
            mode: CryptoJS.mode.CBC,
            padding: CryptoJS.pad.Pkcs7,
            iv: v59
        });
        return v61.toString(CryptoJS.enc.Utf8);
    }

其中全局变量：

    let v58 = CryptoJS.enc.Utf8.parse("xxxxxxxxoooooooo");   // 16字节密钥
    let v59 = CryptoJS.enc.Utf8.parse("0123456789ABCDEF");   // 16字节 IV

**加密参数总结** ：

  * 算法：AES-128-CBC
  * 密钥：  ` xxxxxxxxoooooooo  ` （ASCII 字符串）
  * 初始向量：  ` 0123456789ABCDEF  `
  * 填充：PKCS7
  * 输入：十六进制编码的密文
  * 输出：解密后的 UTF-8 字符串 → 再  ` JSON.parse  ` 得到对象

### Python 还原

    from Crypto.Cipher import AES
    import json

    def decrypt_r(hex_ciphertext: str):
        key = b"xxxxxxxxoooooooo"# 必须是16字节
        iv  = b"0123456789ABCDEF"
        cipher = AES.new(key, AES.MODE_CBC, iv)
        ciphertext = bytes.fromhex(hex_ciphertext)
        plain_padded = cipher.decrypt(ciphertext)
        # 去除 PKCS7 填充
        pad_len = plain_padded[-1]
        if pad_len < 1 or pad_len > 16:
            raise ValueError("Invalid padding")
        plain = plain_padded[:-pad_len]
        return json.loads(plain.decode('utf-8'))

    # 示例
    r_hex = "8b1f3c4d5e6a7b8c9d0e1f2a3b4c5d6e..."# 实际从响应中获取
    data = decrypt_r(r_hex)
    print(data)

## 五、完整请求模拟（Python 集成）

将以上三个环节串联，即可构造完整的加密请求并解密响应：

    import hashlib
    import time
    import requests
    from Crypto.Cipher import AES
    from Crypto.Util.Padding import unpad
    import json
    import os

    BASE_URL = "https://www.mashangpa.com"
    PROBLEM_ID = 7
    AES_KEY = b"xxxxxxxxoooooooo"
    AES_IV = b"0123456789ABCDEF"
    SALT = "xialuo"# 用于生成 m
    FIXED = "xxoo"# 用于生成 x

    def load_cookie() -> str:
        """从 cookie.txt 加载完整的 Cookie 字符串，如 'session_id=xxx'"""
        path = os.path.join(os.path.dirname(__file__), "cookie.txt")
        if os.path.exists(path):
            with open(path, 'r', encoding='utf-8') as f:
                for line in f:
                    line = line.strip()
                    if '=' in line and not line.startswith('#'):
                        return line
        print("[!] 未找到有效 cookie.txt，将不携带 session_id")
        return None

    def generate_m() -> tuple:
        """生成请求头 m: MD5('xialuo' + 毫秒时间戳)"""
        ts = int(time.time() * 1000)
        m = hashlib.md5(f"{SALT}{ts}".encode()).hexdigest()
        return m, ts

    def generate_x(m: str) -> str:
        """生成 URL 参数 x: SHA256(m + 'xxoo')"""
        return hashlib.sha256(f"{m}{FIXED}".encode()).hexdigest()

    def decrypt_response(encrypted_hex: str) -> dict:
        """AES-CBC 解密响应中的 r 字段"""
        cipher = AES.new(AES_KEY, AES.MODE_CBC, iv=AES_IV)
        encrypted_bytes = bytes.fromhex(encrypted_hex)
        decrypted = unpad(cipher.decrypt(encrypted_bytes), AES.block_size)
        return json.loads(decrypted.decode('utf-8'))

    def fetch_page(session: requests.Session, page: int) -> dict:
        """请求单页数据并解密"""
        m, ts = generate_m()
        x = generate_x(m)

        headers = {
            "m": m,
            "ts": str(ts),
            "User-Agent": "Mozilla/5.0"
        }
        params = {
            "page": page,
            "x": x
        }

        url = f"{BASE_URL}/api/problem-detail/{PROBLEM_ID}/data/"
        resp = session.get(url, headers=headers, params=params)
        resp.raise_for_status()

        data = resp.json()
        if "r" not in data:
            raise ValueError(f"响应缺少 r 字段: {data}")
        return decrypt_response(data["r"])

    def main():
        session = requests.Session()

        cookie = load_cookie()
        if cookie:
            session.headers["Cookie"] = cookie
            print(f"[+] 已加载 Cookie: {cookie}")
        else:
            print("[!] 未找到 cookie.txt，请先登录并写入 session_id")
            print("    格式: session_id=你的值")
            return

        # 先访问一下页面建立会话（可选）
        session.get(f"{BASE_URL}/problem-detail/{PROBLEM_ID}/")

        total_sum = 0
        all_numbers = []

        for page in range(1, 21):
            decrypted = fetch_page(session, page)
            current_array = decrypted.get("current_array", [])
            if not isinstance(current_array, list):
                print(f"[!] 第{page}页返回的数据不是数组: {current_array}")
                print("[!] 可能 Cookie 已过期，请重新登录更新 cookie.txt")
                return

            page_sum = sum(current_array)
            total_sum += page_sum
            all_numbers.extend(current_array)
            # 输出数组的具体值（而不是长度）
            print(f"第{page:2d}页: 数组 = {current_array}, 本页和 = {page_sum}, 累计 = {total_sum}")

        print(f"\n{'='*50}")
        print(f"总计页数: 20, 元素总数: {len(all_numbers)}")
        print(f"所有元素之和(答案): {total_sum}")
        print(f"{'='*50}")
        return total_sum

    if __name__ == "__main__":
        main()

## 六、注意事项与经验总结

  1. **密钥硬编码风险**
本例中 AES 密钥和 IV 均明文写在 JS 中，这是严重的安全缺陷。实际对抗中可轻松提取。

  2. **时间戳有效期**
服务端可能会校验  ` ts  ` 的时间范围（如 ±5 分钟），模拟时务必使用  ` int(time.time()*1000)  ` 获取实时毫秒戳。

  3. **参数组合一致性**
` m  ` 和  ` x  ` 的生成顺序必须严格遵循：  ` m = MD5(xialuo+ts)  ` →  ` x = SHA256(m+xxoo)
` ，否则服务端校验失败。

  4. **加密算法组合**
前端使用 MD5 + SHA256 + AES-CBC，强度中等。若需更高安全性，可引入 RSA 或动态密钥协商。

  5. **调试技巧**

     * 轻度混淆的代码可借助  ` JS Deobfuscator  ` 快速还原
     * 调用栈能帮你快速定位核心 JS 文件
     * 在反混淆后的代码中搜索  ` headers.m  ` 、  ` &x=  ` 等关键字，快速找到加密点
