## 一、为什么要学AST？

先说说JS逆向是干啥的。简单讲，就是研究别人写的JS代码，搞清楚它是怎么工作的。比如你想知道某个网站的登录加密是怎么实现的，或者某个App的请求参数是怎么生成的。

但是，很多网站为了保护代码，会对JS进行  **混淆** 。混淆后的代码长这样：

    var _0x4b8c=['\x63\x6f\x6e\x73\x6f\x6c\x65','\x6c\x6f\x67','\x48\x65\x6c\x6c\x6f\x20\x57\x6f\x72\x6c\x64'];(function(_0x4f78e1,_0x4b8c2a){var _0x5a3b4c=function(_0x2d8f07){while(--_0x2d8f07){_0x4f78e1['push'](_0x4f78e1['shift']());}};_0x5a3b4c(++_0x4b8c2a);}(_0x4b8c,0x1f3));var _0x5a3b=function(_0x4f78e1,_0x4b8c2a){_0x4f78e1=_0x4f78e1-0x0;var _0x5a3b4c=_0x4b8c[_0x4f78e1];return _0x5a3b4c;};function hello(){console[_0x5a3b('0x0')](_0x5a3b('0x1'));}

这谁看得懂？但是，不管怎么混淆，这段代码最终还是要被浏览器执行。而浏览器在执行之前，会先把代码解析成一种结构化的数据——这就是  ** AST（抽象语法树）
** 。

换句话说，  **AST是代码的“骨骼”**
。混淆只是改变了代码的外貌（变量名、结构），但骨骼不会变。我们通过操作AST，就能把混淆后的代码“正骨”，还原成容易理解的样子。

## 二、AST到底是什么？

想象一下，我们有一句话：“小明喜欢小红”。人类能立刻理解，但计算机不行。计算机需要把这句话拆成主谓宾：主语（小明）、谓语（喜欢）、宾语（小红）。这个拆分结果，就是一个树状结构。

JS代码也一样。比如下面这行代码：

    let a = 1 + 2;

把它拆成树，大概是这样：

    VariableDeclaration (变量声明)
      ├─ kind: "let"
      └─ declarations:
          └─ VariableDeclarator
                ├─ id: Identifier (变量名) -> "a"
                └─ init: BinaryExpression (二元表达式)
                      ├─ operator: "+"
                      ├─ left: Literal -> 1
                      └─ right: Literal -> 2

这棵树就是  **AST** 。每个方框叫一个  **节点（Node）** 。节点有不同的类型，比如  ` NumericLiteral  `
（数字字面量）、  ` Identifier  ` （标识符）等等。

有了这棵树，我们就可以精准地找到代码中的某个部分，然后修改它。比如把数字1改成100，或者把变量名a改成b。

## 三、动手准备：安装工具

要操作AST，我们需要一些工具。最常用的是Babel全家桶，它是专门用来处理JS代码的。

新建一个文件夹，打开终端（命令行），执行以下命令：

    npm init -y
    npm install @babel/parser @babel/traverse @babel/generator @babel/types

  * ` @babel/parser  ` ：把JS代码解析成AST
  * ` @babel/traverse  ` ：遍历AST，找到我们要改的节点
  * ` @babel/generator  ` ：把修改后的AST重新生成JS代码
  * ` @babel/types  ` ：辅助判断节点类型和创建新节点

装好后，我们就可以开始写第一个AST脚本了。

## 四、第一个AST脚本：把数字123变成456

我们来写一个简单的脚本：把代码中所有出现的数字  ` 123  ` ，都改成  ` 456  ` 。

创建一个文件  ` demo.js  ` ，输入以下代码：

    const parser = require('@babel/parser');
    const traverse = require('@babel/traverse').default;
    const generate = require('@babel/generator').default;

    // 原始代码
    const code = `
    let a = 123;
    let b = 456;
    let c = 123 + 456;
    `;

    // 1. 解析成AST
    const ast = parser.parse(code);

    // 2. 遍历AST
    traverse(ast, {
    // 当我们遇到一个数字字面量节点时，执行这个函数
    NumericLiteral(path) {
        // 如果这个数字的值是123
        if (path.node.value === 123) {
          // 把它改成456
          path.node.value = 456;
        }
      }
    });

    // 3. 生成新的代码
    const output = generate(ast).code;

    console.log('原始代码：');
    console.log(code);
    console.log('修改后：');
    console.log(output);

保存后，在终端运行：

    node demo.js

你会看到输出：

    原始代码：
    let a = 123;
    let b = 456;
    let c = 123 + 456;

    修改后：
    let a = 456;
    let b = 456;
    let c = 456 + 456;

神奇吧！所有123都变成了456。

### 代码解析

  * ` parser.parse(code)  ` ：把字符串代码变成AST。
  * ` traverse(ast, visitor)  ` ：遍历AST，visitor是一个对象，里面定义了我们要对哪些节点做什么操作。比如这里我们定义了对  ` NumericLiteral  ` 节点的处理函数。
  * ` path.node.value  ` ：获取当前节点的值。  ` NumericLiteral  ` 节点有个属性  ` value  ` ，就是数字本身。
  * ` generate(ast).code  ` ：把修改后的AST变回代码字符串。

就这么简单！

## 五、还能做什么？

学会了基础操作，我们可以做更多事情：

### 1\. 简化常量计算

把  ` 1 + 2  ` 直接算成  ` 3  ` 。思路是：遇到  ` BinaryExpression  `
节点，如果左右两边都是数字，就计算结果，然后用数字节点替换整个表达式。

    const t = require('@babel/types');

    traverse(ast, {
    BinaryExpression(path) {
        const { left, right, operator } = path.node;
        if (t.isNumericLiteral(left) && t.isNumericLiteral(right)) {
          let result;
          switch (operator) {
            case '+': result = left.value + right.value; break;
            case '-': result = left.value - right.value; break;
            case '*': result = left.value * right.value; break;
            case '/': result = left.value / right.value; break;
            default: return;
          }
          path.replaceWith(t.numericLiteral(result));
        }
      }
    });

运行后，  ` let c = 123 + 456;  ` 会变成  ` let c = 579;  ` 。

### 2\. 给变量统一改名

比如把所有变量名  ` a  ` 改成  ` myVar  ` 。但要小心，不要改了关键字或函数名。这里只是演示思路：

    traverse(ast, {
      Identifier(path) {
        if (path.node.name === 'a' && !path.isReferencedIdentifier()) {
          path.node.name = 'myVar';
        }
      }
    });

## 六、新手学习AST的好帮手：AST Explorer

有一个超好用的在线工具：  AST Explorer  [1]  。你可以在左边写JS代码，右边实时看到对应的AST结构。

强烈建议新手多去玩玩这个工具：

  * 写一段简单代码，看看AST长什么样。
  * 试着点开不同的节点，观察它们的属性。
  * 写一段混淆代码，看看AST和普通代码有什么区别。

你会很快熟悉各种节点类型。

## 七、总结

AST抽象语法树，其实就是代码的结构化表示。通过操作AST，我们可以像做手术一样精准地修改代码，让混淆代码现出原形。

新手入门AST，不需要一下子学太深。从最简单的数字修改开始，慢慢尝试常量计算、变量改名。多动手，多观察AST
Explorer，很快你就能自己写出有用的AST插件。

记住：  **AST不可怕，可怕的是不去动手** 。现在就去试试吧！
