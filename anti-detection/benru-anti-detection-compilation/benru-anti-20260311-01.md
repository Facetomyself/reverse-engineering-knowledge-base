## 二、最基础的图文验证码

就是那种给你一张图，上面几个扭曲的字母数字，让你输入。这种现在虽然少了，但老系统还在用。

### 我以前怎么搞的？

最早用 Tesseract，那叫一个折腾——要调二值化、去噪点、训练字库… 搞半天识别率还不到 60%。后来发现有大佬写了个  **ddddocr**
（带带弟弟OCR），真·三行代码跑起来：

    import ddddocr
    ocr = ddddocr.DdddOcr()  # 全局初始化，别每次都new
    with open('code.png', 'rb') as f:
        img_bytes = f.read()
    res = ocr.classification(img_bytes)
    print(res)

这个库最爽的是不用配环境，pip 完就能跑。而且支持中文、字母数字混合，甚至能过滤颜色。比如那种红字验证码：

    res = ocr.classification(img_bytes, color_filter_colors=['red'])

**踩坑提醒** ：别每次识别都 new 一个对象，放全局初始化就行，不然慢得你想哭。

## 三、滑块验证码——拦路虎本虎

滑块应该是目前最常见的。极验、腾讯的滑块，缺口位置千奇百怪，背景也越来越花。

### 第一步：找到缺口在哪儿

以前用 OpenCV 模板匹配，背景一变就歇菜。后来 ddddocr 也出了检测功能，直接返回缺口坐标：

    det = ddddocr.DdddOcr(det=True)  # 开启目标检测模式
    with open('bg.jpg', 'rb') as f:
        bg_bytes = f.read()
    bboxes = det.detection(bg_bytes)  # 返回 [[x1,y1,x2,y2], ...]
    if bboxes:
        x1, y1, x2, y2 = bboxes[0]   # 取第一个目标（缺口）
        print(f'缺口坐标：({x1}, {y1}) - ({x2}, {y2})')
    else:
        print('没找到缺口')

**说明** ：  ` detection  ` 返回的是坐标列表，每个元素是  ` [x1, y1, x2, y2]  `
。如果图片里只有一个缺口，取第一个即可。他还提供了两种专门的滑块验证码处理算法，可以去  `
https://github.com/sml2h3/ddddocr  ` 瞧瞧。

### 第二步：怎么拖才像人？

光有坐标没用，直接 drag_and_drop 大概率被识别成机器。得模拟人类拖动——先快后慢，中间抖一抖。

我写了个简单的轨迹生成函数（实际应用中可能需要更复杂的物理模型，但入门够用了）：

    import random
    import time

    def get_track(distance):
        """生成模拟人类的拖动轨迹"""
        track = []
        current = 0
        mid = distance * 3 / 4# 加速阶段占前3/4
        while current < distance:
            if current < mid:
                move = random.randint(2, 5)   # 加速阶段每次移动大一点
            else:
                move = random.randint(1, 2)   # 减速阶段移动小一点
            current += move
            track.append(move)
        return track

配合 Selenium 拖动：

    from selenium.webdriver import ActionChains

    slider = driver.find_element_by_xpath('//div[@class="slider"]')
    ActionChains(driver).click_and_hold(slider).perform()
    for x in get_track(300):  # 300 是缺口距离（需要自己计算）
        ActionChains(driver).move_by_offset(x, random.randint(-1, 1)).perform()
        time.sleep(0.01)
    ActionChains(driver).release().perform()

**说点实际的**
：就算有轨迹，有时候还是会被识别。后来我发现，拖动之前先让鼠标在页面上随便晃晃，成功率会高不少。可能是验证码会检测鼠标有没有移动轨迹。

## 四、点选验证码——让人头大

“请依次点击汽车、手机、红绿灯”。这种最烦，因为要识别图片里的物体，还要按顺序点。

### 咋整？

ddddocr 的目标检测可以返回图中每个物体的坐标，但注意它  **只返回坐标，不返回物体的标签**
。所以你需要先知道每个坐标对应什么物体——这通常需要另外的识别步骤（比如把每个目标截图下来用 OCR 或分类模型识别），或者直接用打码平台。

如果想自己试，可以这样拿到所有目标的位置：

    det = ddddocr.DdddOcr(det=True)
    with open('click_captcha.png', 'rb') as f:
        img_bytes = f.read()
    bboxes = det.detection(img_bytes)  # 返回多个目标的坐标
    for i, bbox in enumerate(bboxes):
        print(f'目标{i+1}: {bbox}')  # 例如 [23, 45, 78, 120]

然后你需要根据提示（比如“点击汽车”）判断哪个坐标是汽车，这通常需要借助别的模型或人工标注。所以我个人建议：  **点选验证码直接用打码平台**
，几毛钱一次，省心省力。

**如果你要自己搞** ：可以把每个目标裁剪出来，用 ddddocr 的  ` classification  `
识别上面的文字（如果目标上有文字），或者用其他图像分类库（如  ` cnocr  ` ）。但代码量会翻几倍，且准确率不一定高。

## 五、旋转验证码——让你把图转正

有些网站喜欢用旋转验证码，比如一个倒着的风景图，让你转正。

### 怎么解？

我的建议是：  **遇到旋转验证码，直接上打码平台** 。这种验证码出现频率低，自己写代码调试半天，可能还不如花两毛钱让人工给你转。

如果你真的想自己搞，可以搜一下开源的角度检测模型（比如  ` rotate-captcha-cnn  ` ），或者用 OpenCV
的特征点匹配估算角度，但难度较大，不推荐新手尝试。

## 六、语音验证码——念一段数字

有些网站为了照顾视障用户，会出语音验证码。爬虫遇到这个其实挺少见，但如果真遇到了，可以用语音识别。

### 我试过的方法

用百度语音识别 API（有免费额度）：

    from aip import AipSpeech

    APP_ID = '你的 App ID'
    API_KEY = '你的 Api Key'
    SECRET_KEY = '你的 Secret Key'

    client = AipSpeech(APP_ID, API_KEY, SECRET_KEY)

    with open('voice.wav', 'rb') as f:
        speech = f.read()
    res = client.asr(speech, 'wav', 16000, {'dev_pid': 1536})  # 1536 表示普通话+英语
    print(res['result'][0])  # 输出识别的数字

记得把语音文件转成合适的格式，MP3 要先转 WAV。可以用 ffmpeg 转：

    ffmpeg -i voice.mp3 voice.wav

## 七、无感验证码——防不胜防

这种最阴，不弹窗，但是后台偷偷分析你的鼠标轨迹、浏览器指纹、请求频率。你要是直接上 selenium，大概率被识别。

### 怎么伪装？

  1. **隐藏 webdriver 特征** ：用 selenium-stealth 库

    pip install selenium-stealth

    from selenium import webdriver
    from selenium_stealth import stealth

    options = webdriver.ChromeOptions()
    options.add_argument("--disable-blink-features=AutomationControlled")
    driver = webdriver.Chrome(options=options)

    stealth(driver,
            languages=["zh-CN", "zh"],
            vendor="Google Inc.",
            platform="Win32",
            webgl_vendor="Intel Inc.",
            renderer="Intel Iris OpenGL Engine",
            fix_hairline=True)

  2. **模拟人类操作** ：别每次都一秒钟点完，中间加随机延迟，偶尔把鼠标移开再回来。

  3. **控制请求频率** ：太快肯定被封，太慢也不现实。可以设置随机间隔，比如 2-5 秒。

## 八、短信/邮箱验证码

这种一般是登录时用的，需要接收验证码。

### 我的做法

  * **接码平台** ：sms-activate.org、5sim.net，花几块钱买个临时手机号，收验证码。
  * **邮箱自动读取** ：如果验证码发到自己邮箱，可以用 IMAP 提取。

邮箱读取示例：

    import imaplib
    import email

    conn = imaplib.IMAP4_SSL('imap.qq.com')
    conn.login('<redacted-email>', 'password')  # 注意密码是授权码，不是登录密码
    conn.select('INBOX')
    _, data = conn.search(None, 'UNSEEN')  # 查找未读邮件
    for num in data[0].split():
        _, msg_data = conn.fetch(num, '(RFC822)')
        msg = email.message_from_bytes(msg_data[0][1])
        # 提取邮件正文（需要处理编码）
        if msg.is_multipart():
            for part in msg.walk():
                if part.get_content_type() == 'text/plain':
                    body = part.get_payload(decode=True).decode('utf-8')
                    break
        else:
            body = msg.get_payload(decode=True).decode('utf-8')
        # 从body里用正则提取验证码...
        print(body)
    conn.close()
    conn.logout()

**提醒一下** ：用邮箱自动化要小心，别把自己账号搞封了。而且验证码属于敏感信息，别乱存。

## 九、最后

验证码这东西，本质是防脚本的，但爬虫和反爬本来就是道高一尺魔高一丈。遇到难的，别硬刚，换个思路——能用现成库就用现成库，能用打码平台就花钱，时间也是成本。
