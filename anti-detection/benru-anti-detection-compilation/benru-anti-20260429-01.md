## 一、技术原理

点选验证码的破解，本质上是  **两个模型接力** 。

**第一棒：YOLO 检测（找位置）** YOLO 负责在验证码图片中框出所有文字，并区分“目标字”（  ` target  `
，题干要你点的）和“干扰字”（  ` char  ` ，用来混淆视线的）。它只输出坐标和类别标签，不识字。关键后处理有两步：先丢掉置信度低于 0.25
的框（被干扰线遮挡的字通常在 0.3~0.6），再用 NMS 合并重叠框。验证码里的字挨得很近，NMS 的 IoU 阈值要从默认的 0.45 降到
0.3，否则会把紧邻的不同字误并成一个。

**第二棒：孪生网络匹配（排顺序）** 目标字找到了，但不知道谁先谁后。孪生网络不直接分类（几千个汉字类别太多，且同字不同字体差异巨大），而是做
**图像匹配** ：把每个目标字的截图和字库里的参考字逐一比对，输出 0~1 的相似度。相似度 ≥0.7 判定为同一字，≤0.3
明确排除，中间灰色地带通常意味着字体不匹配。训练时用  **对比损失** 让同字靠近、异字远离，后期混入 20%
形近字（困难负样本）能硬拉几个百分点的准确率。

**串联流程** ：YOLO 先找出所有  ` target  `
的位置，孪生网络再把它们和字库里的字一一配对，配对成功的位置按字库顺序输出，就是点击顺序。换个验证码类型需要重训，根源在于字体一变，孪生网络的匹配阈值就全塌了。

### 2.3 识别不准？排查清单

优先级  |  原因  |  怎么办
---|---|---
⭐最高  |  截图尺寸/质量不对  |  查 CSS 缩放，人工看截图是否模糊
⭐⭐⭐  |  NMS 参数不合适  |  漏检→降 IoU（0.2）；多检→提置信度（0.4）
⭐⭐  |  字体变化  |  看匹配分数，全卡 0.4~0.6 就是字体不匹配
⭐  |  分辨率过低  |  YOLO 最低要求 320×320

## 三、如果预训练模型不适用你的目标网站

当你换了一个验证码类型（不同网站、不同字体），预训练模型大概率会失效。这时需要自己训练。

### 3.1 准备数据（最关键的一步）

你需要至少  **200-300 张** 目标网站的验证码截图，存到  ` dataset/images/  ` 下。

然后用标注工具给每张图打标签。推荐  LabelImg  [1]  ：

    pip install labelImg
    labelImg

**标注规则：**

  * 题干让你点的字 → 标签选  ` target  `
  * 背景干扰字 → 标签选  ` char  `
  * 保存格式选  **YOLO** （会生成  ` .txt  ` 文件）

标注完的目录结构：

    dataset/
    ├── images/
    │   ├── captcha_001.jpg
    │   ├── captcha_002.jpg
    │   └── ...
    └── labels/
        ├── captcha_001.txt
        ├── captcha_002.txt
        └── ...

### 3.2 训练 YOLOv5 检测模型

    git clone https://github.com/ultralytics/yolov5
    cd yolov5
    pip install -r requirements.txt

    # 创建 data.yaml
    echo "path: ../dataset
    train: images
    val: images
    nc: 2
    names: ['char', 'target']" > data.yaml

    # 开始训练
    python train.py --data data.yaml --weights yolov5s6.pt --img 640 --batch 16 --epochs 100

训练完，模型在  ` runs/train/exp/weights/best.pt  ` 。

### 3.3 训练孪生网络匹配模型

这一步比较繁琐，核心是  **构建正负样本对** 。简化做法：

  1. 用训练好的 YOLO 模型把所有验证码图片里的 target 和 char 框全裁出来
  2. 同一个字的截图放一个文件夹，不同字的放不同文件夹
  3. 随机组正样本对（同文件夹内两张图）和负样本对（不同文件夹各取一张）
  4. 用  Siamese-pytorch  [2]  训练

这里提供一个自动化脚本思路（需自行调整路径）：

    import os, random, shutil
    from itertools import combinations

    # 假设所有文字截图按字分类存在 crops/ 下
    # crops/面/  crops/筋/  crops/青/  ...
    word_dirs = os.listdir('crops')

    pairs_dir = 'siamese_data'
    os.makedirs(f'{pairs_dir}/positive', exist_ok=True)
    os.makedirs(f'{pairs_dir}/negative', exist_ok=True)

    # 正样本对：同一文件夹内随机两张
    for wd in word_dirs:
        imgs = os.listdir(f'crops/{wd}')
        if len(imgs) < 2: continue
        for a, b in random.sample(list(combinations(imgs, 2)), min(20, len(imgs))):
            shutil.copy(f'crops/{wd}/{a}', f'{pairs_dir}/positive/{wd}_{a}')
            shutil.copy(f'crops/{wd}/{b}', f'{pairs_dir}/positive/{wd}_{b}')

    # 负样本对：不同文件夹各取一张（加入形近字）
    for _ in range(500):
        w1, w2 = random.sample(word_dirs, 2)
        img1 = random.choice(os.listdir(f'crops/{w1}'))
        img2 = random.choice(os.listdir(f'crops/{w2}'))
        shutil.copy(f'crops/{w1}/{img1}', f'{pairs_dir}/negative/{w1}_{img1}')
        shutil.copy(f'crops/{w2}/{img2}', f'{pairs_dir}/negative/{w2}_{img2}')

然后按 Siamese-pytorch 的格式训练即可。

## 四、回到原点

从 B 站那张“面筋”验证码，到程序完美点击提交，全过程就三层：

    YOLO 检测层 → 定位 target 区域
    孪生网络匹配层 → 按字库排序
    模拟点击层 → 贝塞尔曲线 + 随机延迟

每一层单独看都不复杂，难的是串联后每个参数彼此牵连——YOLO 的 IoU
阈值决定孪生网络的输入质量，孪生网络的匹配阈值影响漏配率。做这件事的爬虫工程师，说到底就是个  **超参数调优师** 。

调参的真正价值，在于把抽象数字和验证码的物理世界对应起来：
**IoU 阈值 ≈ 文字间距，置信度 ≈ 干扰线遮挡程度，匹配阈值 ≈ 字体差异容忍度** 。
当你看一张 B 站验证码时，脑中浮现的不再是“面”和“筋”，而是一个个边界框和相似度矩阵——你就真的出师了。

这套流程是通用的——B 站、京东、知网，只要点选验证码结构类似，步骤完全一样，区别只在于换训练数据。
