## 指纹检测：爬虫工程师的噩梦

首先，让我们直面现实：现代网站的反爬虫技术已经进化到了令人惊叹的程度。

“浏览器指纹”不再是科幻概念，而是每个爬虫工程师必须面对的日常挑战。

让我给大家看一个典型的检测场景：

    # 普通爬虫面临的检测风险
    from selenium import webdriver

    driver = webdriver.Chrome()
    driver.get("https://target-site.com")

    # 你以为你在隐身访问，其实网站可能正在收集：
    # - WebGL指纹
    # - Canvas指纹
    # - 字体列表
    # - 浏览器插件信息
    # - 硬件特性
    # 等等数十种标识特征

当这些指纹特征组合在一起，你的爬虫就像在聚光灯下裸奔——即使使用代理IP轮换，服务器也能轻易识别：“嘿，又是那个脚本！”

## stealth.js：从Node.js世界传来的火种

stealth.js最初是为Puppeteer设计的，它的目标简单而明确：  **让自动化浏览器看起来像真人浏览器** 。

核心原理是通过一系列精心设计的补丁，消除自动化工具留下的“机器痕迹”。

    // Node.js中的典型用法
    const puppeteer = require('puppeteer-extra')
    const StealthPlugin = require('puppeteer-extra-plugin-stealth')

    puppeteer.use(StealthPlugin())

    const browser = await puppeteer.launch({ headless: false })

但问题是，我们Python爬虫工程师怎么办？难道要为了这个工具转投Node.js阵营？

当然不！经过多次实战测试，我找到了将stealth.js集成到Python爬虫的几种有效方法。

### 方案一：Playwright + stealth.js补丁
![](https://mmbiz.qpic.cn/mmbiz_svg/Q3auHgzwzM5kvd9l79xW5OOahq1lQMkay47KPdUnlicRCYM9eCmnWyFYpvA3HknNycmaOudUQuOD3LCoN5qs0KRiaoSoSAKNdibkeebBgQM3u3yUrPNdtbY2A/640?wx_fmt=svg&from=appmsg)

Playwright是我目前最推荐的方案，它天生对CDP（Chrome DevTools
Protocol）有良好支持，让我们可以注入JavaScript补丁。

    from playwright.sync_api import sync_playwright
    import json

    defapply_stealth_playwright(page):
        """应用stealth.js补丁到Playwright页面"""
        stealth_js = """
        // 这里放置stealth.js的核心代码
        // 通常包括：
        // 1. 覆盖navigator.webdriver属性
        // 2. 修复plugins长度
        // 3. 处理language属性
        // 4. 修补chrome运行时
        Object.defineProperty(navigator, 'webdriver', {
            get: () => undefined
        });

        // 更多stealth.js补丁...
        """

        page.add_init_script(stealth_js)

        # 额外的CDP会话调用
        cdp_session = page.context.new_cdp_session(page)
        cdp_session.execute_cdp_command('Page.addScriptToEvaluateOnNewDocument', {
            'source': stealth_js
        })

    with sync_playwright() as p:
        browser = p.chromium.launch(headless=False)
        context = browser.new_context()
        page = context.new_page()

        # 关键步骤：应用反检测补丁
        apply_stealth_playwright(page)

        page.goto('https://bot.sannysoft.com')
        page.screenshot(path='stealth-test.png')
        browser.close()

### 方案二：Selenium + CDP直接调用

对于坚守Selenium的团队，我们也可以通过CDP直接调用stealth.js补丁：

    from selenium import webdriver
    from selenium.webdriver.chrome.service import Service
    from selenium.webdriver.chrome.options import Options

    defenable_stealth_mode(driver):
        """为Selenium驱动启用stealth模式"""
        stealth_js_path = "./stealth.min.js"

        withopen(stealth_js_path, 'r') as f:
            stealth_script = f.read()

        # 通过CDP执行stealth.js
        driver.execute_cdp_cmd(
            "Page.addScriptToEvaluateOnNewDocument",
            {"source": stealth_script}
        )

        # 额外补丁：覆盖webdriver属性
        driver.execute_script("""
            Object.defineProperty(navigator, 'webdriver', {
                get: () => undefined
            });
        """)

    # 配置Chrome选项
    chrome_options = Options()
    chrome_options.add_argument("--disable-blink-features=AutomationControlled")
    chrome_options.add_experimental_option("excludeSwitches", ["enable-automation"])
    chrome_options.add_experimental_option('useAutomationExtension', False)

    driver = webdriver.Chrome(options=chrome_options)

    # 关键：在访问任何页面前应用stealth补丁
    enable_stealth_mode(driver)

    # 验证效果
    driver.get("https://bot.sannysoft.com")

## 性能与稳定性的平衡

使用stealth.js需要考虑性能开销：

    def benchmark_stealth_impact():
        """测试stealth.js对性能的影响"""
        import time

        test_cases = [
            ("无stealth", lambda: create_basic_driver()),
            ("有stealth", lambda: create_stealth_driver()),
        ]

        for name, creator in test_cases:
            start_time = time.time()

            for _ inrange(10):

## 未来展望与替代方案

stealth.js虽然强大，但也要关注新兴技术：

  1. **Playwright原生stealth模式** ：Playwright团队正在开发更完善的反检测支持
  2. **Undetected ChromeDriver** ：专门针对Selenium的反检测增强
  3. **自定义浏览器构建** ：修改Chromium源代码彻底移除自动化特征

## 总结

stealth.js为Python爬虫工程师提供了一个强大的反检测武器，但记住：

在爬虫与反爬虫的永恒博弈中，stealth.js是我们手中的重要筹码。掌握它，但不要依赖它；使用它，但要理解它背后的原理。毕竟，最好的反检测技术，是深入理解检测原理后，量身打造的解决方案。
